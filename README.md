# 通过LMT登录设备
介绍如何通过LMT登录统一网关，以及执行命令行的操作方法。

## 前提条件
在使用LMT工具之前，请通过验证文件签名检查软件包的完整性。

> 说明：
> 在单网口模式下，您可以通过SCU单板（U1911/U1960/U1981）或者SMCU单板（U1980）上的网口0或网口1登录设备。
> 在双网口和三网口模式下，您必须通过SCU单板（U1911/U1960/U1981）或者SMCU单板（U1980）上的网口0才能登录设备。

## 背景信息
用户可以使用LMT工具登录设备，通过命令树执行查看和配置命令，还可以进行告警管理、信令收集、版本升级、日志收集和离线操作等。

通过LMT登录统一网关可以分为View模式，Config模式和Debug模式（统一网关的V200R003C50SPC600及之后版本支持Debug模式）。
- View模式
用户缺省登录的都是View模式，在该模式下用户可以执行show命令和CLI一般命令（如clear、help、list、exit等）。View模式下的命令提示符为“>”。
- Config模式
Config模式需要从View模式切换后进入，在该模式下用户除可以执行View模式下能够执行的所有命令外，还可以执行config命令和设备维护命令（如reboot、switch board等）。Config模式下的命令提示符为“#”。
- Debug模式：在Config模式下，输入super进入Debug模式，可以支持发送Debug命令。Debug模式下的显示界面的命令提示符为[%eSpace U1980(debug)]$。
LMT工具支持32位和64位的Windows 7、Windows 8、Windows 10（V200R003C30SPC300及之后版本支持Windows 10）。在64位操作系统上运行LMT，如果出现兼容性问题，请到微软官方网站下载并安装兼容32位程序的补丁，链接：http://www.microsoft.com/zh-cn/download/confirmation.aspx?id=5582。


> **说明**：
LMT使用SSH/Telnet协议连接eSpace U1900设备，最大可以同时连4台。

## 登录设备
本节以增加一台eSpace U1960设备为例，说明通过LMT增加和登录设备的操作方法，实际的IP地址以修改后的为准。
1. 双击运行“IPT_LMT”文件夹下的“IPT_LMT.exe”文件。
系统弹出“本地维护终端”，
2. 右键单击“设备树”，在弹出的快捷菜单中选择“新增设备”。
3. 增加统一网关设备，单击“确定”。
   > 说明：
     - 如果您要新增的U1911/U1960/U1981地址类型为IPV6，请先在“IP地址”后面的下拉列表选择“ipv6”，再输入IP地址。
     - 您也可以在菜单栏中选择“设备管理 > 新增设备”增加设备。

4. 右键单击“设备树”下需要连接的设备，在弹出的快捷菜单中选择“连接设备”。
5. 系统弹出“连接设备”对话框，配置连接参数。

   > 说明：
    - 输入“用户名”、“View模式密码”和“Config模式密码”（默认值分别为“admin”、“Change_Me”和“Change_Me”）。如果只需要登录View模式，则无需输入“Config模式密码”。
    - 登录统一网关的协议分为Telnet和SSH，默认只能选择“启用SSH加密”，因为Telnet不是安全的通信协议，默认关闭。如何开启Telnet协议请参见通过CLI登录设备的“使用Telnet连接”章节。
    - 首次登录后，为了保证帐号安全，请尽快修改密码，并使用复杂度高的密码，若不修改，会存在帐号被盗用的风险。密码修改方法请参见如何修改设备登录密码。
    - 密码修改后，请妥善保存。如果忘记，密码设置方法请参见设备登录密码忘记了怎么办。

6. 单击“确定”，系统进入命令树配置界面。
